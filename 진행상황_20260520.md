## 데이터셋 구성

### M3FD (기본 데이터셋)
- 이미지 쌍: **4,200장** (RGB + IR)
- 해상도: 1024×768 (주)
- 클래스: 6개 — People, Car, Bus, Motorcycle, Lamp, Truck
- 총 박스: 34,407개
- Train/Val: 3,360 / 840 (8:2, seed=42)
- 포맷: VOC XML → YOLO 변환 완료

### FLIR ADAS Aligned (추가 데이터셋)
- 이미지 쌍: **5,142장** (RGB + IR)
- 해상도: 640×512
- 클래스: 4개 — car, person, bicycle, dog
- 총 박스: 40,860개
- Train/Val: 4,126 / 1,013
- M3FD 매핑: person→People(0), car→Car(1), bicycle/dog 제외
- 포맷: COCO JSON → YOLO 변환 완료

### 통합 데이터셋 (M3FD + FLIR)

| M3FD | 3,360 | 840 |

## 해결한 기술적 문제 (Windows 한글 경로)

Windows 환경에서 한글/특수문자(`★`) 경로로 인해 다수의 인코딩 문제 발생 및 해결:

| 문제 | 파일 | 해결 |
|---|---|---|
| cp949 txt 파일 읽기 | datasets.py:869,884 | `open(..., encoding='utf-8')` |
| 슬래시/백슬래시 혼재 | datasets.py:908 | `replace('\\', '/')` |
| 라벨 경로 매핑 오류 | datasets.py:img2label_paths | `/rgb/`→`/labels/` 변환 추가 |
| 캐시 경로 백슬래시 | datasets.py:964,983 | `replace('\\', '/')` |
| cv2.imread 유니코드 불가 | datasets.py:1340,1370 | `np.fromfile` + `cv2.imdecode` |
